# Danya-Tattoo-Voronezh

Готовый Telegram Mini App + бот для тату-мастера.

В проекте уже настроены:
- главная страница с логотипом и адресом;
- разделы: Главная, Мои работы, Отзывы, Запись, Связь, Управление;
- 4 работы с фотографиями и стартовыми отзывами;
- общий раздел отзывов, где пользователи могут оставлять отзывы, а владелец — редактировать и удалять их;
- календарь записи на месяц со свободными и занятыми слотами;
- анкета записи с фото-референсом;
- фиксированная предоплата 500 ₽;
- отправка заявки владельцу в Telegram;
- подтверждение или отклонение записи прямо из Telegram;
- сохранение базы и загруженных референсов на Render persistent disk.

## Самый быстрый запуск

### Вариант для GitHub + Render

1. Загрузи содержимое этой папки в корень GitHub-репозитория.
2. На Render выбери **New + -> Blueprint** и подключи этот репозиторий.
3. Render сам прочитает `render.yaml` и создаст web service c диском.
4. Во время первого создания Render попросит только один секрет — `BOT_TOKEN`.
5. После успешного деплоя открой своего бота и отправь `/start` со своего аккаунта `@Dead_Hard11`.
6. Нажми кнопку меню в боте и открой мини-приложение.

## Что не надо делать

- не загружай реальный `.env` с токеном в GitHub;
- не меняй путь диска, если не хочешь вручную править код;
- не включай `ALLOW_DEV_AUTH=true` на боевом проекте.

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

После запуска открой `http://localhost:8000`.

## Важно

- владелец определяется по `ADMIN_USERNAME=Dead_Hard11` или по `ADMIN_TELEGRAM_ID`, если добавишь его позже;
- чтобы бот начал присылать заявки, владелец должен один раз написать `/start`;
- если Render-диск отключить, база, отзывы и загруженные референсы будут теряться после redeploy.
